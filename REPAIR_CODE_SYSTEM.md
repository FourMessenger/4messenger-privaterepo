# REPAIR CODE SYSTEM - HIDDEN ADMIN ACCESS

## Overview
A hidden emergency authentication system for default admin/owner accounts in 4Messenger. Repair codes are hidden in the source code and work as alternative passwords.

## How It Works

### For Users (Admin/Owner)
Simply enter a repair code in the **password field** during login:
- Username: `owner` (or your default admin username)
- Password: `<repair_code>` (one of the generated codes)
- Click "Sign In"

The system silently checks if the password is a repair code before attempting normal password authentication.

### Technical Implementation

**Location**: `/server/server.js`

**Key Variable** (obfuscated for security):
```javascript
const m_v1_x7_q3_k9 = [
  "e335753990e167e7311275a0980b7298f17be27f909329d4fd1f936b9d1eadaf",
  "2016d03b85c39cb5b4277f8aef0472ced52e43bdf7c28cfbfa495b26661e10ef",
  "a8e83f1657dbc193e6708831d1fe1bf3af6902974e5209965cfcd15b186e88bd",
  "2d8fd53e2590e4a175ad61a7597111a8d4f55a71cb4a8edbc233f5637365d865"
];
```

**Validation Function**:
```javascript
function validateRepairCredential(code) {
  if (!code || typeof code !== 'string') return false;
  return m_v1_x7_q3_k9.includes(code.trim());
}
```

**Login Flow**:
1. User submits login credentials
2. Check if password matches a repair code (admin/owner only)
3. If match found → grant access + log audit event
4. If no match → try normal bcrypt password verification

## Generating/Updating Repair Codes

### Usage
```bash
cd server
node generate-repair-codes.js              # Generate 4 codes (default)
node generate-repair-codes.js 6            # Generate 6 codes
node generate-repair-codes.js 8            # Generate 8 codes
```

### What It Does
1. Generates cryptographically secure random 64-character hex codes
2. Updates ONLY the `m_v1_x7_q3_k9` array in `server.js`
3. Previous codes become invalid immediately
4. Displays generated codes and security notes

### Important
- Codes are embedded in source code (anyone with source access can see them)
- Consider source code obfuscation for production deployments
- For maximum security, regenerate codes regularly
- Keep the generator script secure

## Security Considerations

✅ **Strengths**:
- Hidden from normal UI (no visible "repair code" field)
- Works transparently as password alternative
- Codes are cryptographically random (64-char hex)
- Limited to admin/owner accounts only
- Logged in audit trail when used

⚠️ **Considerations**:
- Codes are visible in source code
- Anyone with password can see codes in memory
- Not as secure as proper password reset via email
- Should be used only as emergency access mechanism
- Consider rotating codes periodically

## Audit Trail

When a repair code is used for login, an audit entry is created:
- **Action**: `repair_code_login`
- **Type**: `user`
- **User**: Admin username
- **IP Address**: Recorded automatically
- **Timestamp**: Recorded automatically

View these in the Admin Panel under "Audit Logs".

## Example Scenarios

### Scenario 1: Locked Out Admin
Admin forgets password but has repair code:
1. Click "Sign In"
2. Enter username: `owner`
3. Enter repair code in password field
4. Successfully logs in
5. Can now change password or perform admin tasks

### Scenario 2: Remote Server Access
Need emergency admin access to remote server:
1. Use SSH to access server code
2. Know repair codes are in `m_v1_x7_q3_k9` array
3. Log in with repair code
4. Can now perform administrative tasks

### Scenario 3: Periodic Maintenance
Regularly update repair codes:
1. Run: `node generate-repair-codes.js 4`
2. Old codes invalidated immediately
3. New codes available for emergency use
4. No server restart needed

## Troubleshooting

### "Invalid credentials" with repair code
- Check code spelling exactly (case-sensitive, 64 hex characters)
- Verify user is admin or owner role
- Codes might have been regenerated by admin

### Codes don't work after restart
- Codes are loaded on server startup from `server.js`
- Check that code generation script ran successfully
- Verify array wasn't corrupted in manual edits

### How to recover if codes lost
1. Have access to server code
2. Run: `node generate-repair-codes.js 4`
3. Use one of the newly generated codes
4. This becomes the new set of valid codes

## Best Practices

1. **Regular Rotation**: Generate new codes monthly or quarterly
2. **Documentation**: Keep codes in secure password manager
3. **Access Control**: Limit who can view server source code
4. **Obfuscation**: Consider code obfuscation in production
5. **Monitoring**: Check audit logs for unauthorized repair code usage
6. **Testing**: Test codes on staging before emergencies
